# Security Policy

If you believe you have found a security vulnerability in PID Piper, please
report it responsibly.

## Reporting a vulnerability

- **Email:** pid-piper@shumil.in
- Please include:
  - A clear description of the issue and impacted versions.
  - Steps to reproduce (proof-of-concept if available).
  - Any logs, crash dumps, or screenshots that help diagnose the issue.

> Please do **not** open public GitHub issues for security vulnerabilities.

## Supported versions

Security fixes are provided for the latest released version.

| Version | Supported |
| --- | --- |
| Latest | ✅ |
| Older versions | ❌ |

## Disclosure timeline

We aim to acknowledge reports within **7 days**.

When a report is confirmed, we will coordinate a fix and public disclosure.
As a guideline, we target a maximum of **90 days** from acknowledgement to
public disclosure, but this may vary based on severity and complexity.

## Release integrity & provenance

Public releases include supply-chain artifacts (e.g. build provenance
attestations, SBOMs, and Sigstore/cosign signatures) so users can verify that
downloaded binaries were produced by the official release workflow.

## Bug bounties

We do not currently offer a paid bug bounty program.
